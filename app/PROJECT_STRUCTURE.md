# BBNL IPTV – LG webOS TV App — Complete Project Structure & Explanation

## Overview

This is a **React 19** web application packaged as an **LG webOS TV app** (`.ipk`). It is an IPTV (Internet Protocol Television) client for BBNL (Bharat Broadband Network Limited). The app runs on LG Smart TVs and is controlled via the LG Magic Remote control. It streams live TV channels, shows OTT app listings, allows language-wise channel browsing, handles user authentication via OTP, and reports playback data (TRP).

---

## Tech Stack

| Technology | Purpose |
|---|---|
| React 19 | UI framework |
| React Router v7 (MemoryRouter) | Client-side routing (no hash/browser history — required for webOS) |
| Material UI (MUI) v7 | UI component library |
| Zustand v4 | Global state management |
| Axios | HTTP API calls |
| hls.js v1 | HLS (m3u8) live stream playback |
| react-window | Virtualized list rendering (performance) |
| LG webOS Luna APIs | TV hardware: device ID, network info, MAC, Magic Remote sensor |

## Detailed File-by-File Explanation

---

### `src/index.js` — App Entry Point

Mounts the React app into `<div id="root">` in `public/index.html`. No `StrictMode` wrapper (intentionally removed to avoid double-render issues on webOS).

---

### `src/App.js` — Root Component & Router

**What it does:**
- Wraps the entire app in `MemoryRouter` (required for webOS — no browser URL bar)
- Manages `isAuthenticated` state from `localStorage`
- On app load, calls `initializeWebOSEnvironment()` and `initializeMagicRemoteUIStability()`
- After login, checks `checkAppLock()` API — if service is locked, overlays `<ServiceLocked />`
- Preloads all lazy-loaded routes using `requestIdleCallback` after authentication
- Defines all routes with authentication guards (`Navigate to /login` if not authenticated)

**Route Map:**

| Route | Component | Notes |
|---|---|---|
| `/` | Redirect | → `/home` or `/login` |
| `/bbnl-video` | `BbnlVideo` | Intro video before login |
| `/login` | `PhoneNumberOtp` | OTP login screen |
| `/home` | `Home` | Main dashboard |
| `/live-channels` | `LiveChannels` | Full channel browser |
| `/languagechannels` | `LanguageChannels` | Language selection |

| `/favorites` | `Favorites` | Favorites (coming soon) |

| `/settings` | `Setting` | Settings + logout |

**Performance:** All routes use `lazyWithPreload()` — a wrapper around `React.lazy()` that also attaches a `.preload()` method for eager loading in the background.

### `src/Api/Deviceinformaction/LG-Devicesinformaction.jsx` — Device Info Hook

**`useDeviceInformation()`** — Custom React hook.

Fetches all TV hardware details in parallel using `Promise.all()`:
- **Public IP** (IPv4 + IPv6) — from external IP service
- **Private IP** — from Luna network API
- **MAC Address** (wired + wifi) — from Luna hardware API
- **Model Name / Firmware** — from Luna system properties API
- **Hardware Device ID** — from `deviceStorage.js` (LGUDID → MAC → UUID fallback)

Returns `{ publicIPv4, privateIPv4, deviceId, wiredMac, modelName, firmwareVersion, loading, error }`.

Used in: `App.js` (app lock check), `LoginOtp.jsx` (display on login screen), `Setting.jsx` (device info page).

### `src/Api/OAuthentication-Api/LoginOtpApi.jsx` — OTP Login with Device Info

**`sendOtpWithWebOSInfo(phone)`** — Collects webOS device info (model, MAC, IP) then calls `POST /login` with full device payload. Returns `{ success, otp, custdet, deviceInfo }`.

**`checkAppLock(payload)`** — Posts to `/applock`. If `response.data.body.lock === "true"`, the entire app is replaced by a full-screen `ServiceLocked` overlay. Prevents unauthorized access.

## Global State Stores (`src/Global-storage/`)

All stores use **Zustand** for state management. `HomeStore.jsx` provides shared infrastructure used by all other stores.

---

### `HomeStore.jsx` — Base Infrastructure

Exports shared utilities used by ALL other stores:
- **`postJson(url, payload)`** — Wrapper around `fetch()` with default headers
- **`postForm(url, payload)`** — x-www-form-urlencoded variant
- **`buildAuthPayload(phone, options)`** — Builds standard auth request body
- **`nowMs()`** — High-resolution timestamp using `performance.now()`

---

### `AuthStore.jsx` — Authentication State

Zustand store managing OTP login flow:
- `sendOtp(phone, options)` — Calls `POST /login`, stores OTP from response
- `resendOtp(phone, options)` — Calls `POST /loginOtp` for resend
- `verifyOtp(phone, otp)` — Verifies OTP locally or via API
- State: `{ isLoading, error, success }`

---

### `LiveChannelsStore.jsx` — Channels & Categories Cache

Zustand store with **multi-level cache system**:
- Categories cached for **2 hours** (`CATEGORIES_TTL_MS`)
- Channels cached for **30 minutes** per `userid|mobile|grid` key
- `fetchCategories(payload, options)` — Loads + caches channel groups
- `fetchChannels(payload, options)` — Loads + caches channels by grid/key
- `getChannelByNumber(payload, value)` — Fast channel lookup by LCN number
- Used in: `LiveChannels.jsx`, `LivePlayer.jsx`, `ChannelsSidebar.jsx`

---

### `LivePlayersStore.jsx` — Language Cache

Zustand store for language list:
- Cached per `userid|mobile` key, TTL **2 hours**
- `fetchLanguages(payload)` — Loads and caches from `/chnl_langlist`
- Used in: `Home.jsx`, `LiveChannels.jsx`, `ChannelsView.jsx`

---

### `ChannelsSearchStore.jsx` — Home Ads Cache

Zustand store for IPTV ads:
- Cached per `userid|mobile|area|type|client|srctype` key, TTL **10 minutes**
- `fetchAds(payload)` — Loads home/stream ads
- Used in: `HomeAds.jsx`

---

### `FeedbackStore.jsx` — Feedback State

- `submitFeedback(payload)` — Posts to `/feedback` API
- State: `{ isSubmitting, error, successMessage }`
- Used in: `Feedback.jsx`

---

### `LogineOttp.jsx` — App Version Cache

- `fetchAppVersion(payload)` — Loads and caches app version from `/appversion`, TTL **2 hours**
- Used in: `Setting.jsx`

## Authentication Flow (`src/OAuthenticate/`)

---

### `bbnl.jsx` — Intro Video Screen

- Plays `/Asset/BBNL.mp4` fullscreen on app launch
- On video `ended` or `error` — automatically navigates to `/login`
- Entry route: `/bbnl-video`

### `LoginOtp.jsx` — OTP Login (Two-Step)

**Step 1 — Mobile Number Entry:**
- User enters 10-digit mobile number
- Sends request to `POST /login` with full webOS device info (model, MAC, IP)
- Displays Gateway IP and TV Model name on screen
- Shows `RegisterNumber` error if number is not registered
- On success, stores `serverOtp` from API response and moves to Step 2

**Step 2 — OTP Verification:**
- User enters 6-digit OTP
- Compares with `serverOtp` received from Step 1
- On match: stores `userId`, `userPhone` in localStorage, sets `isAuthenticated = true`, navigates to `/home`
- On mismatch: shows `ValidOTP` error component
- 30-second resend timer with countdown

**Background:** Dynamically fetched login background image. Falls back to hardcoded URL.
**Logo:** Dynamically fetched from `/fofitv_logo` API.

## Common Components (`src/Atomic-Common-Componenets/`)

### `useMagicRemote.js` — Magic Remote Hooks

Two hooks:

**`useMagicRemote(options)`:**
- Connects to LG's Luna MRCU service: `luna://com.webos.service.mrcu`
- Tracks Magic Remote pointer coordinates (x, y)
- Supports webOS TV 24+ sensor2 API and legacy fallback
- Returns `{ coordinates, sensorData, isReady, setSensorInterval, resetQuaternion }`

**`useEnhancedRemoteNavigation(items, options)`:**
- Manages keyboard/remote navigation across a list or grid of items
- Supports `orientation: 'vertical' | 'horizontal' | 'grid'`
- Tracks `focusedIndex` and `hoveredIndex` (Magic Remote pointer hover)
- Provides `getItemProps(index)` — returns `{ onMouseEnter, onClick, 'data-focused', 'data-hovered' }`
- Calls `onSelect(index)` callback when user presses OK/Enter on focused item

## Sub-Components (`src/Atomic-Module-Componenets/`)

### `StreamPlayer.jsx` — HLS Video Player

Core video player:
- Uses `hls.js` for HLS (`.m3u8`) stream playback
- Sets custom HTTP headers on every segment request: `X-Package-Id`, `X-App-Name`, `X-Platform`
- Handles stall detection (10 second timeout → auto-restart)
- Automatic network retry (up to 3 times)
- Buffer cleanup every 2 minutes
- Full media key support via `MEDIA_KEY_MAP` and `MEDIA_KEYCODE_MAP`
- Detects native HLS support (Safari/webOS native player) vs. hls.js

### `ChannelCard.jsx` — Channel Preview Card

### `deviceStorage.js` — Hardware Device ID

Manages permanent device identification with **3-tier priority**:

1. **LGUDID** (LG Unique Device ID from Luna API) — best, truly permanent
2. **MAC Address hash** — hardware-based backup
3. **localStorage UUID** — last resort (can change if localStorage is cleared)

**`getDeviceUUID()`** — Returns a stable device ID following the priority chain.  
**`getFormattedDeviceId()`** — Returns `TV-{id}` formatted string.

Purpose: Enforces **one phone number = one device** subscription rule.

---

### `magicRemoteUIStability.js` — UI Layout Lock

Prevents layout shifts when Magic Remote pointer moves around the screen:
- `lockUIForMagicRemote()` — Locks `#root` element (fixed position, no overflow)
- `disableFocusScaling()` — Injects CSS to prevent `transform: scale()` on focus
- `getStableCoordinates(coords, rect)` — Clamps pointer coordinates to container bounds
- Called from `App.js` on startup via `initializeMagicRemoteUIStability()`
